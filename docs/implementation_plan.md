# 実施計画書：標識問題システムの全面刷新

[設計書](file:///Users/kk/development/menkyo/docs/sign_system_design.md)に基づき、既存の `signs.json` を使用せず、200問の高品質な道路標識データを新規に作成する計画です。

## ユーザーの確認が必要な事項

> [!IMPORTANT]
> 既存の `signs.json` は無視し、より堅牢なスキーマを持つ `signs_new.json` を新規作成します。また、正確性を期すため、ウィキメディア・コモンズから高品質なSVG/PNG画像をダウンロードします。

> [!WARNING]
> 本計画の実行により、約100枚の画像ファイルが `/Users/kk/development/menkyo/data/signs_new/` にダウンロードされます。ディスク容量とインターネット接続をご確認ください。

## 提案する変更内容

### 1. 画像の取得
- **スクリプトによる自動取得**: 公開されている公式標識番号に基づき、ウィキメディア・コモンズから約100種類の標識画像を自動取得します。
- **フォーマット変換**: Web環境での表示の安定性を考慮し、必要に応じてSVGからPNGへの変換を行います。

### 2. データ作成 (`signs_new.json`)
- **新スキーマの導入**: 設計書で定義した階層構造を持つJSONを生成します。
- **200問の作成**: 各標識に対して2〜3問を作成し、以下の点に重点を置きます。
    - **形式**: 全問 **○×（True/False）形式** で統一します（既存チャプターとの整合性維持）。
    - 正確な定義の確認
    - 実際の試験に出る典型的な「ひっかけ」パターン
    - 二種免許レベルの細かなニュアンス（バス・タクシー視点のひっかけ等）

### 3. 配置と管理
- **Chapter区分**: 新しい標識問題一式は、既存の構成に合わせて **「Chapter 99」** として管理します。
- **ディレクトリ**: 画像は `/Users/kk/development/menkyo/data/signs_new/` に格納します。

### 4. クリーンアップ
- 新システムの検証完了後、旧 `signs.json` を削除するか、バックアップとして残すかを決定します。

## 確定した仕様 (User Feedback)

- **画像形式**: 原則として **PNG** を使用します。SVGから変換する際、実際の標識の表記（フォントや細部）が損なわれないよう細心の注意を払います。
- **問題形式**: 全問 **○×形式**。
- **配置**: **Chapter 99**。
