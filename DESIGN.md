# 普通二種免許 学科試験対策アプリ — 設計書

> **最終更新:** 2026-04-13
> **ステータス:** v1.0

---

## 1. プロジェクト概要

### 1.1 目的

普通自動車第二種運転免許の学科試験（95問・100点満点・90点合格）に合格するための、
網羅的なWebベースの学習アプリを提供する。

### 1.2 目標

- **問題数:** 1,500問（目標範囲: 1,000〜2,000問）
- **カバレッジ:** 試験の全11章を網羅
- **ひっかけ対策:** 混同しやすい標識ペア・断定表現の罠を重点的に出題
- **標識コツ:** 覚え方・語呂合わせを解説に組み込み

### 1.3 ターゲットユーザー

- 普通2種免許の学科試験を受験する人
- タクシー・ハイヤー・バス運転手を目指す人
- 一種免許は取得済みで二種免許への追加取得を考えている人

## 3. ひっかけ問題・覚え方コツの組み込み方針

### 3.1 ひっかけパターン一覧

| パターン | 例 | 正解傾向 |
|---------|-----|---------|
| 断定表現の罠 | 「必ず〜」「絶対に〜」「いかなる場合でも〜」 | x（例外ありのことが多い） |
| 安全配慮表現 | 「注意する」「確認する」「徐行する」 | o（安全行動は正解のことが多い） |
| 危険行動 | 「加速する」「急ハンドル」「急ブレーキ」 | x（危険行動は不正解） |
| 逆の意味の標識 | 車両通行止め vs 駐車禁止（白地 vs 青地） | 混同注意 |
| 数字の罠 | 定期点検「1ヶ月」（x）正しくは「3ヶ月」 | 数字を変えて出題 |
| 条件の有無 | 「要件を満たせば」の有無で正逆が変わる | 条件に注目 |

### 3.2 標識のひっかけ10ペア（必ずペアで出題）

| ペア | 違い | ひっかけポイント |
|------|------|-----------------|
| 車両通行止め vs 駐車禁止 | 白地 vs 青地 | 色が逆！ |
| 車両進入禁止 vs 車両通行止め | 片側のみ vs 区域全体 | 範囲の違い |
| 車両通行止め vs 通行止め | 歩行者OK vs 歩行者もNG | 対象の違い |
| 一方通行 vs 指定方向以外進行禁止 | 長方形 vs 丸形 | 形で見分け |
| 左折可 vs 一方通行 | 白地に青矢印 vs 青地に白矢印 | 色が逆！ |
| 車両横断禁止 vs 二輪以外の自動車通行止め | 図柄が似ている | 正答率低 |
| 指定方向進行禁止 vs 進行方向別通行区分 | 名前も図柄も似ている | 混同注意 |
| 専用通行帯 vs 優先通行帯 | その車しか vs 他車も通行可 | 「しか」vs「も」 |
| 最高速度 vs 積載制限 | 数字が入る規制標識 | 意味の取り違え |
| 駐車禁止 vs 駐停車禁止 | x印 vs 斜線のみ | 記号の違い |

### 3.3 覚え方コツの例（解説に組み込み）

- **屈曲/屈折/背向**: 屈曲=なだらか、屈折=カクカク、背向=2つ
- **青地 vs 白地**: 青地=ゆるい（駐車だけNG）、白地=ストップ（通行NG）
- **フェード現象**: フェード=フェードアウト=効かなくなる
- **ハイドロプレーニング**: 水にプレーニング=水の上を滑る
- **ベーパーロック**: ベーパー=蒸気=ブレーキ液が沸騰
- **幅員減少**: 「ふくいんげんしょう」と読む。道路の幅が狭くなる警戒標識

## 4. データ構造設計

### 4.1 問題データのJSON形式

各章ごとにJSONファイルを作成する。以下は1問の例：

```json
{
  "id": "001_001",
  "chapter": 1,
  "category": "信号の意味・種類",
  "type": "truefalse",
  "question": "赤信号のとき、すべての車は交差点に入ってはならない。",
  "answer": true,
  "explanation": "赤信号はすべての車に対して交差点への進入を禁止します。ただしすでに交差点内にいる車はそのまま進めます。",
  "tags": ["信号", "基本"],
  "difficulty": 1,
  "trickType": null
}
```

### 4.2 スキーマ定義

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|:----:|------|
| `id` | string | o | "章番号3桁_通番3桁" 形式（例: 001_001） |
| `chapter` | integer | o | 章番号（1,3,5,6,7,8,9,10,11） |
| `category` | string | o | サブカテゴリ名 |
| `type` | string | o | "truefalse" 固定 |
| `question` | string | o | 問題文 |
| `answer` | boolean | o | 正解（true=o、false=x） |
| `explanation` | string | o | 解説文。ひっかけ問題は「【ひっかけポイント】」を含む |
| `tags` | string[] | o | 検索・フィルタ用タグ |
| `difficulty` | integer | o | 難易度 1（基礎）2（標準）3（難） |
| `trickType` | string/null | o | ひっかけの種類。nullなら通常問題 |

### 4.3 ひっかけ種類（trickType）一覧

| trickType | 説明 |
|-----------|------|
| `null` | 通常問題 |
| `"absolute_expression"` | 断定表現の罠（「必ず」「絶対に」） |
| `"similar_signs"` | 混同しやすい標識ペア |
| `"wrong_number"` | 数字の間違い |
| `"missing_condition"` | 条件の欠落 |
| `"reversed_meaning"` | 意味が逆転 |

### 5.3 HTMLからの読み込み方法

`index.html` のJavaScriptで `fetch()` を使用して各JSONを読み込む：

```javascript
async function loadAllQuestions() {
  const files = ['ch01','ch03','ch05','ch06','ch07','ch08','ch09','ch10','ch11'];
  const allQuestions = [];
  for (const f of files) {
    const res = await fetch(`data/${f}.json`);
    const data = await res.json();
    allQuestions.push(...data);
  }
  return allQuestions;
}
```

> **注意:** ローカルファイルアクセスには簡易HTTPサーバーが必要。
> `python3 -m http.server 8000` で起動して `http://localhost:8000` にアクセス。
